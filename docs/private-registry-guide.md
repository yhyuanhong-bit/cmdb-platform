# Private Docker Registry Setup Guide

> For deploying CMDB Platform to multiple internal machines without internet access.

---

## Architecture

```
Developer Machine          Registry Server         Target Machines
(has source code)          (internal network)      (production/test)
                           
  build + push  ──────→   registry:5000   ←────── docker pull
                              │
                           stores all
                           Docker images
```

## Option A: Simple Registry (5 minutes)

Best for: small teams, < 10 machines, no auth needed.

### 1. Choose a Registry Server

Pick any machine on your internal network with:
- Docker installed
- 20GB+ free disk
- Accessible to all target machines (port 5000)

### 2. Start Registry

```bash
docker run -d \
  --name registry \
  --restart always \
  -p 5000:5000 \
  -v /data/registry:/var/lib/registry \
  registry:2
```

Verify: `curl http://localhost:5000/v2/_catalog` should return `{"repositories":[]}`.

### 3. Configure Docker on ALL machines (Registry Server + Target Machines)

Docker rejects HTTP registries by default. Add your registry to the insecure list:

```bash
# Edit or create /etc/docker/daemon.json
cat > /etc/docker/daemon.json << EOF
{
  "insecure-registries": ["REGISTRY_IP:5000"]
}
EOF

# Replace REGISTRY_IP with the actual IP (e.g., 192.168.1.100)
# Then restart Docker
systemctl restart docker
```

### 4. Push Images (from Developer Machine)

```bash
cd /path/to/cmdb-platform
./scripts/push-to-registry.sh REGISTRY_IP:5000/cmdb v1.2.0
```

### 5. Deploy on Target Machine

```bash
# Copy these 3 files to the target machine:
#   - docker-compose.yml
#   - docker-compose.registry.yml (generated by push script)
#   - .env.example → .env

# On target machine:
cp .env.example .env
vim .env  # set passwords
docker compose -f docker-compose.yml -f docker-compose.registry.yml up -d
```

### 6. Update on Target Machine

```bash
docker compose -f docker-compose.yml -f docker-compose.registry.yml pull
docker compose -f docker-compose.yml -f docker-compose.registry.yml up -d
```

## Option B: Harbor (Enterprise)

Best for: larger teams, need Web UI, access control, vulnerability scanning.

### 1. Download Harbor Offline Installer

On a machine with internet:
```bash
curl -LO https://github.com/goharbor/harbor/releases/download/v2.11.0/harbor-offline-installer-v2.11.0.tgz
```

Copy to the Registry Server.

### 2. Install Harbor

```bash
tar xzf harbor-offline-installer-v2.11.0.tgz
cd harbor

# Edit configuration
cp harbor.yml.tmpl harbor.yml
vim harbor.yml
# Set:
#   hostname: harbor.internal (or IP)
#   harbor_admin_password: your-password
#   For HTTP (no TLS): comment out https section

# Install
./install.sh
```

Harbor Web UI: `http://harbor.internal` (admin / your-password)

### 3. Create Project in Harbor

- Login to Harbor Web UI
- Create project: `cmdb` (public or private)

### 4. Push and Deploy

Same as Option A, but use Harbor URL:
```bash
# Login to Harbor first
docker login harbor.internal

# Push
./scripts/push-to-registry.sh harbor.internal/cmdb v1.2.0

# Deploy on target
docker compose -f docker-compose.yml -f docker-compose.registry.yml up -d
```

## Registry for Air-Gapped Environments

If the Registry Server itself has no internet:

### Step 1: On internet-connected machine

```bash
# Pull the registry image itself
docker pull registry:2
docker save registry:2 | gzip > registry-image.tar.gz

# Build CMDB images
./scripts/build-offline-package.sh v1.2.0
```

### Step 2: Copy to Registry Server

```bash
scp registry-image.tar.gz cmdb-platform-offline-v1.2.0.tar.gz user@registry-server:/opt/
```

### Step 3: On Registry Server

```bash
# Load and start registry
docker load < registry-image.tar.gz
docker run -d --name registry --restart always -p 5000:5000 -v /data/registry:/var/lib/registry registry:2

# Load CMDB images
tar xzf cmdb-platform-offline-v1.2.0.tar.gz
cd cmdb-platform-offline
docker load < images.tar.gz

# Tag and push to local registry
for img in cmdb-core cmdb-frontend ingestion-engine ingestion-worker; do
  docker tag ${img}:latest localhost:5000/cmdb/${img}:v1.2.0
  docker push localhost:5000/cmdb/${img}:v1.2.0
done
```

### Step 4: Target machines pull from Registry Server

Same as before — configure `insecure-registries` and `docker compose up -d`.

## Maintenance

### Check stored images
```bash
curl http://REGISTRY_IP:5000/v2/_catalog
curl http://REGISTRY_IP:5000/v2/cmdb/cmdb-core/tags/list
```

### Disk usage
```bash
# Simple registry
du -sh /data/registry

# Garbage collection (reclaim deleted image space)
docker exec registry bin/registry garbage-collect /etc/docker/registry/config.yml
```

### Backup
```bash
# Backup registry data
tar czf registry-backup-$(date +%Y%m%d).tar.gz /data/registry
```

## Troubleshooting

### "http: server gave HTTP response to HTTPS client"
Docker defaults to HTTPS. Add your registry to `insecure-registries` in `/etc/docker/daemon.json` and restart Docker.

### "connection refused"
Check: `curl http://REGISTRY_IP:5000/v2/` — if fails, verify port 5000 is open and registry container is running.

### "manifest unknown"
The image tag doesn't exist in the registry. Re-push: `./scripts/push-to-registry.sh REGISTRY_IP:5000/cmdb v1.2.0`

### Harbor login fails
Check harbor.yml hostname matches what you're using. For HTTP, ensure the https section is commented out.
