# Phase 4.3 — Sync Envelope HMAC 签名修复报告

**Date:** 2026-04-20
**Source roadmap:** `docs/reports/audit-2026-04-19/REMEDIATION-ROADMAP.md` § 4.3
**Current commit at audit:** `eb72127` (master tip)
**Owner:** platform / sync squad
**Severity:** HIGH (cryptographic integrity → tenant cross-contamination via NATS forgery)

---

## 目标

给 `SyncEnvelope` 加一个**发送端签发、接收端强制验证**的 HMAC-SHA256 签名,把现有
`SHA-256 Checksum(防意外损坏)` 升级成 **真正的消息认证码(防伪造)**。具体:

1. 任何能连到 NATS(`CMDB_SYNC` 流, subject `sync.>`)的进程、镜像、旁路工具,
   无论是否拥有合法 tenant 的 JWT,都**不能**发出一条被 edge agent 接受的 envelope。
   接收端 `Verify` 失败时增加 `cmdb_sync_envelope_rejected_total{reason="bad_signature"}`。
2. 支持**密钥轮换**: 不停机地把生产环境从 key-v1 切到 key-v2,滚动期内双 key 均可验证。
3. 提供 **dual-verify window feature flag**(`CMDB_SYNC_SIGNATURE_ENFORCE=warn|enforce|off`),
   让升级期间历史 in-flight envelope 仍可被接受,避免"一键切换即故障"。
4. 不破坏现有 `VerifyChecksum` — Checksum 仍是 payload 完整性指纹,HMAC 是身份鉴别。

**反目标**: 不做 payload 加密(NATS 层走 TLS), 不做 per-envelope nonce 重放防御
(JetStream dedup + `sync_version` guard 已覆盖), 不引入 KMS/HSM(YAGNI)。

## 现状

### 1. Envelope 结构 `cmdb-core/internal/domain/sync/envelope.go:13-52`

`computeChecksum()`(envelope.go:44-47)只覆盖 `EntityID | Version | Diff` —
**`Source`、`TenantID`、`Action`、`Timestamp` 均不在 checksum 内**。这是当前盲点。

## 威胁模型

### Attacker capability matrix

| Attacker            | Reach NATS? | Schema known? | Has JWT? | Has HMAC key? |
|---------------------|:-----------:|:-------------:|:--------:|:-------------:|
| External (WAN)      | No (VPC)    | Yes (源码)     | No       | No            |
| Compromised edge    | Yes         | Yes           | No       | 取决于密钥分发 |
| Rogue tenant admin  | No          | Yes           | Yes (自租户) | No        |
| Compromised CI/ops  | Yes         | Yes           | 无关     | **Yes**       |
| NATS cluster admin  | Yes         | Yes           | 无关     | 取决于密钥托管 |

签名真正目标是**行 2(compromised edge)和行 3(rogue admin 通过旁路工具)**。
行 4 属于 secret-management 信任边界,超出 HMAC 保护面。

### 攻击场景 — Edge 节点被拿下

1. 攻击者从 `edge-bj-01` pod 拿到 NATS URL + credentials。
2. 构造 envelope:
   ```json
   {"id": "<uuid>", "source": "central",
    "tenant_id": "<victim-tenant>", "entity_type": "user_roles",
    "entity_id": "<admin-user-id>", "action": "update",
    "version": 999999999, "timestamp": "2026-04-20T00:00:00Z",
    "diff": {"role_id": "<super-admin-role>"},
    "checksum": "<sha256(entity:version:diff)>"}
   ```
3. `Publish("sync.<victim>.user_roles.update", ...)`.
4. 中央 / 其它 edge 的 `handleIncomingEnvelope`:
   - `env.Source == "central"` → 不跳过
   - `VerifyChecksum()` **pass**(伪造者同算法)
   - `applyGeneric` 执行 `UPDATE user_roles SET sync_version=$1 WHERE id=$2 AND sync_version<$1`

### 爆炸半径

| 维度                                    | 无签名 | 有签名 |
|-----------------------------------------|:------:|:------:|
| 伪造任意 entity 的 apply                | ✓      | ✗      |
| 跨 tenant 伪造(central 伪造)          | ✓      | ✗      |
| 单 edge 失陷的横向扩散                   | 全 fleet | 仅该节点 |
| Alert 规则静默下架(applyAlertRule 无 version guard, agent.go:303-315) | ✓ | ✗ |
| `user_roles` / 审计表污染               | ✓      | ✗      |

## 设计决策

### D1 算法:HMAC-SHA256 vs Ed25519 vs mTLS

| 选项       | Pros                                              | Cons                                                            |
|------------|---------------------------------------------------|-----------------------------------------------------------------|
| HMAC-SHA256| 对称, 分发简单, `crypto/hmac` stdlib, 零依赖        | 任一节点泄漏即全 fleet 沦陷; 无法识别发送方                      |
| Ed25519    | Per-node 密钥对, 只下发 pubkey, 可识别发送方        | 需 PKI 运维; 签名 64B; 验证 ~50µs vs HMAC ~5µs                  |
| mTLS only  | 不改应用; NATS 原生                                | 证书管理; 只能连接级审计不能 envelope 级; 撤销非即时             |

**选 HMAC-SHA256**。理由:(a) 现有 k8s secret + envFrom 链路零新基建;
(b) fleet 规模 ~10 节点下 `env.Source` + NATS client ID 已足够识别源头;
(c) 预留 `SigAlgorithm` 字段,未来切 Ed25519 仅增字段不破协议。

### D2 密钥轮换:KeyRing(选) vs 单 key 滚动

单 key 滚动需全 fleet 秒级同步切换 — 不现实。KeyRing 显式承载"双 key 共存":

```
CMDB_SYNC_HMAC_ACTIVE_KID=v2
CMDB_SYNC_HMAC_KEY_v1=<old 32-byte hex>   # 仍用于验证
CMDB_SYNC_HMAC_KEY_v2=<new 32-byte hex>   # 用于签 + 验证
```

新 envelope 带 `sig_kid=v2`, 接收端按 kid 定位 key。SOP:
加 v2 → 滚动重启 → 切 ActiveKid=v2 → 滚动重启 → 等 7 天 → 移除 v1 → 滚动重启。

### D3 签名范围 — canonical serialization

签名 payload 必须 canonical, 否则 JSON key 顺序 / 空格差异会让两端 HMAC 不一致。
选**字段拼接 canonical form**(不用 JCS RFC 8785, 对 `Diff` 太重):

```
canonical = kid || "\n" || id || "\n" || source || "\n" || tenant_id || "\n" ||
            entity_type || "\n" || entity_id || "\n" || action || "\n" ||
            version || "\n" || timestamp(RFC3339Nano,UTC) || "\n" ||
            sha256(diff_bytes)_hex
```

**Diff 先 sha256 再入 HMAC** — `json.RawMessage` 在 JetStream 下逐字节透传,
两端看到同一串字节。**TenantID 必入签** — 修补现有 checksum 漏洞(见开放问题 #2)。

### D4 新增字段(向后兼容)

```go
Signature    string `json:"sig,omitempty"`      // HMAC-SHA256(canonical) hex
SigKid       string `json:"sig_kid,omitempty"`  // 支持密钥轮换
SigAlgorithm string `json:"sig_alg,omitempty"`  // 预留, 当前恒为 "HS256"
```

### D5 Fail-closed 切换: feature flag `CMDB_SYNC_SIGNATURE_ENFORCE`

| 值        | 行为                                                                      | 阶段  |
|-----------|---------------------------------------------------------------------------|-------|
| `off`     | 不验不签(兼容旧 edge)                                                    | P0   |
| `warn`    | 发送端签;接收端 verify 失败 **只 warn 不 drop**, metric 照增              | P1 (7天窗口) |
| `enforce` | 接收端 verify 失败 → drop + metric + log                                   | P2   |

## 迁移步骤

### Step 1 — 新增 `sync/signing.go`

```go
package sync

import (
    "crypto/hmac"; "crypto/sha256"; "encoding/hex"
    "errors"; "fmt"; "strconv"; "time"
)

type SignerMode string

const (
    SignerModeOff     SignerMode = "off"
    SignerModeWarn    SignerMode = "warn"
    SignerModeEnforce SignerMode = "enforce"
    SigAlgHS256                  = "HS256"
)

type KeyRing struct {
    ActiveKid string
    Keys      map[string][]byte // kid -> >=32 bytes
}

func NewKeyRing(activeKid string, keys map[string][]byte) (*KeyRing, error) {
    if activeKid == "" { return nil, errors.New("signing: active kid required") }
    if _, ok := keys[activeKid]; !ok {
        return nil, fmt.Errorf("signing: active kid %q missing", activeKid)
    }
    for kid, k := range keys {
        if len(k) < 32 {
            return nil, fmt.Errorf("signing: key %q must be >=32 bytes, got %d", kid, len(k))
        }
    }
    return &KeyRing{ActiveKid: activeKid, Keys: keys}, nil
}

// Canonical — 两端必须哈希一致的字节序列。Diff 先 sha256,避免 JSON 再序列化差异。
func (e *SyncEnvelope) Canonical(kid string) []byte {
    diffHash := sha256.Sum256(e.Diff)
    ts := e.Timestamp.UTC().Format(time.RFC3339Nano)
    parts := []string{
        kid, e.ID, e.Source, e.TenantID, e.EntityType, e.EntityID,
        e.Action, strconv.FormatInt(e.Version, 10), ts,
        hex.EncodeToString(diffHash[:]),
    }
    buf := []byte{}
    for i, p := range parts {
        if i > 0 { buf = append(buf, '\n') }
        buf = append(buf, p...)
    }
    return buf
}

func (r *KeyRing) Sign(e *SyncEnvelope) error {
    key, ok := r.Keys[r.ActiveKid]
    if !ok { return fmt.Errorf("signing: active kid %q not in ring", r.ActiveKid) }
    mac := hmac.New(sha256.New, key)
    mac.Write(e.Canonical(r.ActiveKid))
    e.Signature, e.SigKid, e.SigAlgorithm = hex.EncodeToString(mac.Sum(nil)), r.ActiveKid, SigAlgHS256
    return nil
}

var (
    ErrSigMissing    = errors.New("signature missing")
    ErrSigUnknownKid = errors.New("signature kid unknown")
    ErrSigBadAlg     = errors.New("signature algorithm unsupported")
    ErrSigMismatch   = errors.New("signature mismatch")
)

func (r *KeyRing) Verify(e *SyncEnvelope) error {
    if e.Signature == "" || e.SigKid == "" { return ErrSigMissing }
    if e.SigAlgorithm != "" && e.SigAlgorithm != SigAlgHS256 { return ErrSigBadAlg }
    key, ok := r.Keys[e.SigKid]
    if !ok { return ErrSigUnknownKid }
    got, err := hex.DecodeString(e.Signature)
    if err != nil { return ErrSigMismatch }
    mac := hmac.New(sha256.New, key)
    mac.Write(e.Canonical(e.SigKid))
    if !hmac.Equal(got, mac.Sum(nil)) { return ErrSigMismatch }
    return nil
}
```

```go
// 追加字段
type Config struct {
    // ... 现有字段 ...
    SyncSigningMode      string            // "off" | "warn" | "enforce"
    SyncSigningActiveKid string
    SyncSigningKeys      map[string][]byte // kid -> raw bytes
}

// Load() 末尾
cfg.SyncSigningMode     = envOrDefault("CMDB_SYNC_SIGNATURE_ENFORCE", "off")
cfg.SyncSigningActiveKid = os.Getenv("CMDB_SYNC_HMAC_ACTIVE_KID")
cfg.SyncSigningKeys      = loadSyncHMACKeys()

if cfg.SyncSigningMode != "off" {
    if cfg.SyncSigningActiveKid == "" || len(cfg.SyncSigningKeys) == 0 {
        return nil, fmt.Errorf("SYNC_SIGNATURE_ENFORCE=%s requires active kid + keys", cfg.SyncSigningMode)
    }
    if _, ok := cfg.SyncSigningKeys[cfg.SyncSigningActiveKid]; !ok {
        return nil, fmt.Errorf("active kid %q not in configured keys", cfg.SyncSigningActiveKid)
    }
}

// loadSyncHMACKeys walks os.Environ() for CMDB_SYNC_HMAC_KEY_<KID>=<hex>
func loadSyncHMACKeys() map[string][]byte {
    out := map[string][]byte{}
    const prefix = "CMDB_SYNC_HMAC_KEY_"
    for _, kv := range os.Environ() {
        if !strings.HasPrefix(kv, prefix) { continue }
        eq := strings.IndexByte(kv, '=')
        if eq <= len(prefix) { continue }
        kid := strings.ToLower(kv[len(prefix):eq])
        raw, err := hex.DecodeString(kv[eq+1:])
        if err != nil || len(raw) < 32 {
            fmt.Fprintf(os.Stderr, "WARN: ignoring %s — invalid hex or <32B\n", kv[:eq])
            continue
        }
        out[kid] = raw
    }
    return out
}
```

### Step 4 — Publish 签名 `sync/service.go`

```go
type Service struct {
    pool    *pgxpool.Pool
    bus     eventbus.Bus
    cfg     *config.Config
    nodeID  string
    keyring *KeyRing    // nil 当 SyncSigningMode == "off"
    sigMode SignerMode

func NewService(pool *pgxpool.Pool, bus eventbus.Bus, cfg *config.Config) *Service {
    nodeID := cfg.EdgeNodeID
    if nodeID == "" { nodeID = "central" }
    var kr *KeyRing
    if cfg.SyncSigningMode != string(SignerModeOff) {
        var err error
        kr, err = NewKeyRing(cfg.SyncSigningActiveKid, cfg.SyncSigningKeys)
        if err != nil { zap.L().Fatal("sync signing ring invalid", zap.Error(err)) }
    }
    return &Service{pool: pool, bus: bus, cfg: cfg, nodeID: nodeID,
        keyring: kr, sigMode: SignerMode(cfg.SyncSigningMode)}
}

func (s *Service) onDomainEvent(ctx context.Context, event eventbus.Event, entityType, action string) error {
    // ... 现有查 version + 构造 env 的代码 ...
    env := NewEnvelope(s.nodeID, event.TenantID, entityType, entityID, action, version, event.Payload)

    // Phase 4.3 — 签名。warn / enforce 均签; off 跳过。
    if s.keyring != nil {
        if err := s.keyring.Sign(&env); err != nil {
            zap.L().Error("sync: sign envelope failed", zap.String("id", env.ID), zap.Error(err))
            telemetry.SyncEnvelopeFailed.WithLabelValues(entityType).Inc()
            return nil  // 不发未签的 envelope

    syncSubject := fmt.Sprintf("sync.%s.%s.%s", event.TenantID, entityType, action)
    data, _ := json.Marshal(env)
    return s.bus.Publish(ctx, eventbus.Event{Subject: syncSubject, TenantID: event.TenantID, Payload: data})
}
```

### Step 5 — Subscribe fail-closed 校验 `sync/agent.go`

```go
type Agent struct {
    // ... 现有字段 ...
    keyring *KeyRing
    sigMode SignerMode
}

func (a *Agent) handleIncomingEnvelope(ctx context.Context, event eventbus.Event) error {
    var env SyncEnvelope
    if err := json.Unmarshal(event.Payload, &env); err != nil {
        zap.L().Warn("sync agent: invalid envelope", zap.Error(err))
        return nil
    }
    if env.Source == a.nodeID { return nil }

    // Phase 4.3 — 先验签再校验 checksum 再 apply。
    if a.keyring != nil {
        if verr := a.keyring.Verify(&env); verr != nil {
            reason := signatureRejectReason(verr)
            telemetry.SyncEnvelopeRejected.WithLabelValues(env.EntityType, reason).Inc()
            logFn := zap.L().Warn
            if a.sigMode == SignerModeEnforce { logFn = zap.L().Error }
            logFn("sync agent: signature verify failed",
                zap.String("id", env.ID), zap.String("source", env.Source),
                zap.String("tenant", env.TenantID), zap.String("entity_type", env.EntityType),
                zap.String("kid", env.SigKid), zap.String("reason", reason))
            if a.sigMode == SignerModeEnforce { return nil } // fail-closed
        }
    }

    if !env.VerifyChecksum() {
        telemetry.SyncEnvelopeRejected.WithLabelValues(env.EntityType, "bad_checksum").Inc()
        zap.L().Warn("sync agent: checksum mismatch", zap.String("id", env.ID))
        return nil
    }

    // ... 现有 apply* 分派保持不变 ...
}

func signatureRejectReason(err error) string {
    switch {
    case errors.Is(err, ErrSigMissing):    return "sig_missing"
    case errors.Is(err, ErrSigUnknownKid): return "sig_unknown_kid"
    case errors.Is(err, ErrSigBadAlg):     return "sig_bad_alg"
    case errors.Is(err, ErrSigMismatch):   return "bad_signature"
    default:                               return "sig_unknown"
    }
}
```

### Step 7 — 密钥分发 SOP `docs/runbook/sync-hmac-rotation.md`

1. 生成 `openssl rand -hex 32 > new_key.hex` (32 bytes = 64 hex chars)
2. `kubectl edit secret cmdb-sync-hmac` 加 `CMDB_SYNC_HMAC_KEY_V2=<hex>`, 保留 V1
3. Rolling restart central → edges;此时 central 仍以 V1 签, edges 同时接受 V1+V2
4. 改 `CMDB_SYNC_HMAC_ACTIVE_KID=v2`;先滚 central 再滚 edges
5. 等 7 天(覆盖暂离线 edge 追赶 durable consumer 缓冲)
6. 删 V1 → 滚动重启完毕

## 回滚方案

三档 feature flag 即天然回滚路径:

1. `enforce` → 观察异常高 reject rate
2. `kubectl edit configmap cmdb-core-config` 把 `CMDB_SYNC_SIGNATURE_ENFORCE=warn`
3. `kubectl rollout restart deployment/cmdb-core-*`
4. 如 warn 下仍异常,降到 `off` + 扒日志定位发送端
5. `off` 下纯 checksum 保护继续生效 — 回到 Phase 4.3 之前的基线,不丢数据

关键不变量: checksum 路径始终运行;SQL 层 `sync_version < $N` guard 继续挡老版本。
