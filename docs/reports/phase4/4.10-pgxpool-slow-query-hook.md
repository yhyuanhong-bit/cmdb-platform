# 4.10 pgxpool 慢查询 hook + OTel tracer

**Date:** 2026-04-20
**Commit at design:** `eb72127` (master tip)
**Roadmap reference:** `REMEDIATION-ROADMAP.md` § 4.10 "pgxpool hook: 慢查询 + observability"
**Status:** Ready for implementation — no blocking decisions.

## TL;DR

`internal/platform/database/postgres.go` 的 `NewPool` 当前只配了 `MaxConns / MinConns /
Ping`,**未设置任何 tracer 或 hook**。慢查询只能通过 HTTP 层 `http_request_duration` 反
推,无法定位到具体 SQL。

本设计用 pgx/v5 的 `QueryTracer` interface 自研 `SlowQueryTracer`,阈值默认 **500ms**
(env `CMDB_DB_SLOW_QUERY_MS` 覆盖,clamp [50ms, 10s]),命中时 `zap.Warn` + Prometheus
counter `cmdb_db_slow_queries_total`,外加 `cmdb_db_query_duration_seconds` 直方图。
提供 `ComposeTracers(...)` 入口,Phase 4.6 叠加 otelpgx 时无需再改本文件。

---

## 1. 目标

1. 任何 > 500ms 的 SQL 在生产都有 **zap.Warn + Prometheus counter**,SQL 文本保留占
   位符(不含参数值,防泄漏)。
2. 兼容 OTel:预留 composite 入口,后续叠加 `otelpgx.NewTracer()` 不改 `NewPool` 签
   名。
3. 零侵入 sqlc / handler;所有 `dbgen` / 裸 pool 调用无需改。
4. 运行时可调阈值 via `CMDB_DB_SLOW_QUERY_MS`。
5. 可测试:tracer 本身 unit test + `pg_sleep()` integration test。

## 2. 现状

### 2.1 `postgres.go` 现状

```go
// /cmdb-platform/cmdb-core/internal/platform/database/postgres.go
func NewPool(ctx context.Context, databaseURL string) (*pgxpool.Pool, error) {
    cfg, err := pgxpool.ParseConfig(databaseURL)
    if err != nil { return nil, fmt.Errorf("parse database url: %w", err) }
    cfg.MaxConns = 50
    cfg.MinConns = 5
    pool, err := pgxpool.NewWithConfig(ctx, cfg)
    if err != nil { return nil, fmt.Errorf("create pool: %w", err) }
    if err := pool.Ping(ctx); err != nil { return nil, fmt.Errorf("ping: %w", err) }
    return pool, nil
}
```

- `cfg.ConnConfig.Tracer = nil` → pgx 走 fast-path, 零 instrumentation。
- `go.mod`: `github.com/jackc/pgx/v5 v5.9.1`(>= 5.3 已有 `QueryTracer` 接口)。
- grep `QueryTracer` / `TraceQueryStart` 全仓库: **0 命中**,确认无人已接入。

### 2.2 pgx/v5 的 tracer 接口

pgx 提供 4 种 tracer:`QueryTracer`(Query/QueryRow/Exec)、`BatchTracer`(SendBatch)、
`ConnectTracer`、`CopyFromTracer`。本设计只实现 `QueryTracer`,覆盖 >99% 热路径。
BatchTracer 留给 Phase 4.6 的 otelpgx。

### 2.3 DB 层 observability 现状

HTTP 层有 Prometheus middleware,eventbus 有 publish counter,workflows 有错误抑制
counter。DB 层**是 observability 空洞区**,这是本 phase 优先级虽为 P2 但价值明显的
原因。

## 3. 设计决策

### 3.1 自研 vs otelpgx

| | 自研 `SlowQueryTracer` | otelpgx |
|---|--------------------------|---------|
| 依赖 | 0 新包 | `github.com/exaring/otelpgx` + otel SDK |
| 输出 | zap + Prometheus | OTel span |
| 阈值可配 | 原生 | 需自定义 sampler |
| 实施成本 | 低 | 中(需 collector / tempo) |

**决策:** 本 phase 落自研 tracer + 预留 composite;Phase 4.6 再叠加 otelpgx。

### 3.2 慢查询阈值

| 值 | 评价 |
|---|------|
| 250ms | 过严,cold cache 就命中,日志噪声大 |
| **500ms** | **推荐**,2x 的 HTTP P95 预算 |
| 1s | 过松,用户感知已灾难 |

取 500ms,env `CMDB_DB_SLOW_QUERY_MS` 覆盖,范围 [50, 10000] ms(防止配 0 → 日志海啸,
防止配 >10s → 等价禁用)。

### 3.3 日志脱敏

**绝不打 query 参数值。** pgx 默认不把 args 拼回 SQL,所以只要**不访问 `data.Args`** 即
安全。SQL 文本截到 1024 字符,防止 bulk insert 超长 SQL 撑爆日志。

### 3.4 Sampling

慢查询罕见,不 sample。将来日志噪声过高再引入 token bucket。

### 4.1 新建 `slow_query_tracer.go`

```go
// /cmdb-platform/cmdb-core/internal/platform/database/slow_query_tracer.go
package database

import (
    "context"
    "time"
    "github.com/jackc/pgx/v5"
    "github.com/prometheus/client_golang/prometheus"
    "github.com/prometheus/client_golang/prometheus/promauto"
    "go.uber.org/zap"
)

type tracerCtxKey struct{}
type queryStartData struct {
    startedAt time.Time
    sql       string
}

var (
    dbSlowQueriesTotal = promauto.NewCounterVec(prometheus.CounterOpts{
        Name: "cmdb_db_slow_queries_total",
        Help: "Count of DB queries exceeding the slow-query threshold.",
    }, []string{"op"})

    dbQueryDurationSeconds = promauto.NewHistogramVec(prometheus.HistogramOpts{
        Name:    "cmdb_db_query_duration_seconds",
        Help:    "Histogram of DB query durations (seconds).",
        Buckets: prometheus.ExponentialBuckets(0.001, 2, 14), // 1ms → ~16s
    }, []string{"op"})
)

type SlowQueryTracer struct {
    threshold time.Duration
    logger    *zap.Logger
    maxSQLLog int
}

func NewSlowQueryTracer(threshold time.Duration, logger *zap.Logger) *SlowQueryTracer {
    if threshold <= 0 {
        threshold = defaultSlowQueryMs * time.Millisecond
    }
    if logger == nil {
        logger = zap.L()
    }
    return &SlowQueryTracer{threshold: threshold, logger: logger, maxSQLLog: 1024}
}

func (t *SlowQueryTracer) TraceQueryStart(ctx context.Context, _ *pgx.Conn, data pgx.TraceQueryStartData) context.Context {
    return context.WithValue(ctx, tracerCtxKey{}, queryStartData{startedAt: time.Now(), sql: data.SQL})
}

func (t *SlowQueryTracer) TraceQueryEnd(ctx context.Context, _ *pgx.Conn, data pgx.TraceQueryEndData) {
    start, ok := ctx.Value(tracerCtxKey{}).(queryStartData)
    if !ok {
        return
    }
    dur := time.Since(start.startedAt)
    op := classifyOp(start.sql)
    dbQueryDurationSeconds.WithLabelValues(op).Observe(dur.Seconds())
    if dur < t.threshold {
        return
    }
    dbSlowQueriesTotal.WithLabelValues(op).Inc()

    sql := start.sql
    if len(sql) > t.maxSQLLog {
        sql = sql[:t.maxSQLLog] + "...(truncated)"
    }
    fields := []zap.Field{
        zap.Duration("duration", dur),
        zap.String("op", op),
        zap.String("sql", sql),
        zap.Duration("threshold", t.threshold),
    }
    if data.Err != nil {
        fields = append(fields, zap.Error(data.Err))
    }
    t.logger.Warn("slow db query", fields...)
}
```

### 4.2 Composite tracer (预留 otelpgx 叠加)

`pgxpool.Config` 只接受一个 tracer,所以用 composite 包一层:

```go
// /cmdb-platform/cmdb-core/internal/platform/database/composite_tracer.go
package database

import (
    "context"
    "github.com/jackc/pgx/v5"
)

type compositeQueryTracer struct{ tracers []pgx.QueryTracer }

// ComposeTracers builds a tracer that fans out Start/End events to all
// inputs. Returns nil if all inputs are nil; returns the single tracer
// verbatim if there's only one (fast path).
func ComposeTracers(ts ...pgx.QueryTracer) pgx.QueryTracer {
    filtered := make([]pgx.QueryTracer, 0, len(ts))
    for _, t := range ts {
        if t != nil {
            filtered = append(filtered, t)
        }
    }
    switch len(filtered) {
    case 0:
        return nil
    case 1:
        return filtered[0]
    }
    return &compositeQueryTracer{tracers: filtered}
}

type compositeCtxKey struct{}

func (c *compositeQueryTracer) TraceQueryStart(ctx context.Context, conn *pgx.Conn, data pgx.TraceQueryStartData) context.Context {
    vals := make([]context.Context, len(c.tracers))
    for i, t := range c.tracers {
        vals[i] = t.TraceQueryStart(ctx, conn, data)
    }
    return context.WithValue(ctx, compositeCtxKey{}, vals)
}

func (c *compositeQueryTracer) TraceQueryEnd(ctx context.Context, conn *pgx.Conn, data pgx.TraceQueryEndData) {
    vals, _ := ctx.Value(compositeCtxKey{}).([]context.Context)
    for i, t := range c.tracers {
        sub := ctx
        if i < len(vals) && vals[i] != nil {
            sub = vals[i]
        }
        t.TraceQueryEnd(sub, conn, data)
    }
}
```

### 4.3 `NewPool` 接受 options

```go
// /cmdb-platform/cmdb-core/internal/platform/database/postgres.go
const (
    defaultSlowQueryMs = 500
    minSlowQueryMs     = 50
    maxSlowQueryMs     = 10_000
)

type PoolOption func(*poolConfig)
type poolConfig struct {
    slowQueryThreshold time.Duration
    extraTracers       []pgx.QueryTracer
    logger             *zap.Logger
}

func WithSlowQueryThreshold(d time.Duration) PoolOption { return func(c *poolConfig) { c.slowQueryThreshold = d } }
func WithExtraTracer(t pgx.QueryTracer) PoolOption       { return func(c *poolConfig) { c.extraTracers = append(c.extraTracers, t) } }
func WithLogger(l *zap.Logger) PoolOption                { return func(c *poolConfig) { c.logger = l } }

func NewPool(ctx context.Context, databaseURL string, opts ...PoolOption) (*pgxpool.Pool, error) {
    pc := poolConfig{slowQueryThreshold: resolveThreshold(), logger: zap.L()}
    for _, opt := range opts { opt(&pc) }
    if pc.slowQueryThreshold == 0 { pc.slowQueryThreshold = resolveThreshold() }

    cfg, err := pgxpool.ParseConfig(databaseURL)
    if err != nil { return nil, fmt.Errorf("parse: %w", err) }
    cfg.MaxConns = 50
    cfg.MinConns = 5

    slow := NewSlowQueryTracer(pc.slowQueryThreshold, pc.logger)
    cfg.ConnConfig.Tracer = ComposeTracers(append([]pgx.QueryTracer{slow}, pc.extraTracers...)...)

    pool, err := pgxpool.NewWithConfig(ctx, cfg)
    if err != nil { return nil, fmt.Errorf("create pool: %w", err) }
    if err := pool.Ping(ctx); err != nil { return nil, fmt.Errorf("ping: %w", err) }
    pc.logger.Info("db pool initialized",
        zap.Int32("max_conns", cfg.MaxConns),
        zap.Duration("slow_query_threshold", pc.slowQueryThreshold),
        zap.Int("extra_tracers", len(pc.extraTracers)))
    return pool, nil
}

func resolveThreshold() time.Duration {
    raw := os.Getenv("CMDB_DB_SLOW_QUERY_MS")
    if raw == "" { return defaultSlowQueryMs * time.Millisecond }
    v, err := strconv.Atoi(raw)
    if err != nil { return defaultSlowQueryMs * time.Millisecond }
    if v < minSlowQueryMs { v = minSlowQueryMs }
    if v > maxSlowQueryMs { v = maxSlowQueryMs }
    return time.Duration(v) * time.Millisecond
}
```

## 10. 开放问题

1. **[P2] `classifyOp` 简易 verb 解析是否够用?**
   对 `EXPLAIN` / 前导 `-- comment` 归 `other`。绝大多数 sqlc 生成 SQL 干净,可接受。
   未来需精细再换 pgx 的 parser。

2. **[P2] 是否导出 `pool.Stat()`(`AcquireCount / IdleConns / TotalConns`) 为 Prometheus?**
   独立 collector,本 phase 不做,归 4.6。

3. **[P2] slow query 日志是否加 caller stack frame?**
   用 `runtime.Caller(N)` 上溯到第一个非 `pgx`/`dbgen` 栈帧,定位 10x 更快。Pros: 便于
   定位;Cons: +~10µs CPU + 需 exclude 列表。**推荐 P3 增强**,基础落地后观察。

4. **[P3] query vs exec 分别设阈值?** 大批量 INSERT/UPDATE 天然慢,500ms 误报。可引
   入 `threshold_query=500ms, threshold_exec=2s`。先统一阈值,再看生产误报率。

5. **[P3] Tracer panic 行为?** pgx 是否对 tracer panic 有 recover 未查证。保守: 所有
   tracer 方法只访问 local ctx / zap / prometheus,避免 panic 源。本实现已满足。
