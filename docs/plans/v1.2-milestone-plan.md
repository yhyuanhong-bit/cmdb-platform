# CMDB Platform v1.2 里程碑計劃

> 版本：v1.2.0
> 前置版本：v1.1.0（本次 Session 產出）
> 預估工期：10-12 週
> 狀態：Draft

---

## 一、里程碑目標

完成 Edge 離線同步全功能 + 前端同步管理 UI + 指標拉取實戰化 + E2E 測試覆蓋，使平台具備**多節點分散部署的生產能力**。

---

## 二、工作分解

### Phase A：Edge Sync Phase 2 — 維運同步（3 週）

**前置**：Phase 1 已完成（migration、SyncService、SyncAgent、5 個 API）

| 週次 | 任務 | 交付物 |
|------|------|--------|
| W1 | 工單雙維度狀態機實作 | `execution_status` + `governance_status` 在前後端生效；原 `status` 保持向後相容 |
| W1 | 工單同步邏輯 | Edge 操作 execution_status，Central 操作 governance_status，雙向同步零衝突 |
| W2 | 告警同步 | alert_events 雙向 LWW 同步；alert_rules Central→Edge 單向推送 |
| W2 | 衝突解決前端 UI | 新增 `/system/sync` 頁面：衝突列表、左右對照、一鍵解決、批量操作 |
| W3 | 整合測試 | 模擬 Edge 斷線 → 兩端各改資料 → 恢復 → 驗證同步結果 |

**關鍵檔案**：
- `cmdb-core/internal/domain/maintenance/service.go` — 雙維度狀態機
- `cmdb-core/internal/domain/sync/service.go` — 工單/告警同步邏輯
- `cmdb-demo/src/pages/SyncManagement.tsx` — 新頁面
- `cmdb-demo/src/hooks/useSync.ts` — 新 hook

**驗收標準**：
- [ ] Edge 離線修改工單 execution_status → 恢復後 Central 可見
- [ ] Central 審批 governance_status → Edge 收到更新
- [ ] 同一工單兩端同時修改不同維度 → 零衝突自動合併
- [ ] 衝突 UI 可列出、解決、批量處理 pending 衝突

### Phase C：Edge Sync Phase 4 — 強化與測試（2 週）

| 週次 | 任務 | 交付物 |
|------|------|--------|
| W6 | 壓力測試 | 模擬 14 天離線 → 恢復後完整同步；量測同步耗時和資源消耗 |
| W6 | 混沌測試 | 隨機斷線/恢復（每 5-30 分鐘）持續 24 小時，驗證最終一致性 |
| W7 | Edge 離線 UI | Edge API 同步中返回 503；前端顯示「同步進行中」提示頁 |
| W7 | 文件完善 | Edge 部署指南、運維手冊、故障排除 |

**驗收標準**：
- [ ] 14 天離線後恢復同步 < 30 秒
- [ ] 混沌測試 24 小時後所有節點資料一致
- [ ] Edge 首次啟動 → 自動快照 → 可用，全程無人工操作
- [ ] 部署文件可讓新人獨立完成 Edge 部署

### Phase D：指標拉取實戰化（1 週）

**前置**：P1 已建立 `StartMetricsPuller` 框架（目前只 log）

| 任務 | 交付物 |
|------|--------|
| Prometheus adapter HTTP 拉取 | 每 5 分鐘從 Prometheus `/api/v1/query` 拉取指定 metric → 寫入 metrics 表 |
| 拉取配置 UI | `/system/settings` Adapters tab 加「拉取頻率」和「指標列表」配置 |
| 健康檢查 | adapter 拉取失敗 3 次 → 標記 `status=error`，通知 ops-admin |

**關鍵檔案**：
- `cmdb-core/internal/domain/workflows/subscriber.go` — `pullMetricsFromAdapters` 加入 HTTP 請求
- `cmdb-core/internal/domain/integration/` — Prometheus 回應解析
- `cmdb-demo/src/pages/SystemSettings.tsx` — Adapters tab 增強

**驗收標準**：
- [ ] 配置 Prometheus adapter 後，metrics 表每 5 分鐘自動更新
- [ ] adapter 掛掉後自動告警
- [ ] Energy dashboard 顯示真實 Prometheus 指標

## 四、風險評估

| 風險 | 影響 | 機率 | 緩解措施 |
|------|------|------|---------|
| 工單雙維度改動影響現有功能 | 高 | 中 | 保留原 status 欄位向後相容，雙維度僅同步場景使用 |
| 混沌測試發現資料不一致 | 高 | 中 | 預留 W7 整週做修復；分層佇列設計已降低風險 |
| Prometheus adapter 版本不相容 | 低 | 低 | 使用標準 `/api/v1/query` 端點，相容性好 |
| E2E 測試環境不穩定 | 中 | 中 | Docker Compose 一鍵啟動測試環境 |
| Edge 節點網路環境難以模擬 | 中 | 高 | 用 `tc netem` 模擬延遲/丟包/斷線 |
